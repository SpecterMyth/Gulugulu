# Project Basics

## Core Information
- **Name**: Gulugulu
- **Type**: Desktop virtual pet application
- **Purpose**: Virtual companion that grows alongside user's AI-assisted development work
- **Current Phase**: Initialization (as of 2026-05-20)

## Core Values
1. **Work Integration**: Tightly integrated with developer workflows and AI agent usage
2. **Growth**: Pet evolves based on long-term work patterns and achievements
3. **Social**: Supports cross-user pet interaction and community features
4. **Privacy-first**: All user data stored locally, no mandatory cloud requirements

## Key Constraints
- **Local First**: Primary functionality must work offline without network connection
- **Performance**: Low resource footprint, should not impact development work
- **Cross Platform**: Support Windows, macOS, and Linux
- **Extensible**: Modular architecture to support future feature expansion

## Success Metrics
- User engagement: Daily active usage by developers
- Feature adoption: Usage of growth and social features
- Performance: <5% CPU usage when idle, <100MB memory footprint
