# 咕噜鸭:五行养成 · 核心玩法设计文档(终版 v2)

> **状态:讨论稿。** 本文由多视角设计 + 对抗性评审收敛而成,数值均为讨论起点(非最终值)。文末列出需要你拍板的开放决策。

> 相较 v1 的关键修正:①灵力从"会清零的资源"改为"水位线余额"(贴合代码里 experience 永不下降的事实);②失败机制重做——独一无二的鸭永不被烧,只有可无限补充的 Gen0 普通鸭会被消耗;③砍掉残渣这个第二货币;④诚实标注全 AI 融合是"接口级重构 + 数分钟耗时 + 有真实技术失败率",并把它列为 MVP 头号风险 spike;⑤成功率在点"炼!"瞬间本地结算,失败不调 AI;⑥加入反刷子的真实性节流。

## 一句话愿景

你写的每一行 AI 代码,都在喂养一只桌面上的咕噜鸭;攒够灵力就能抽到金木水火土五行小鸭,把两只鸭亲手炼成一只世界上独一无二的新鸭——你的编程史,长成了一整个咕噜家族。

**一种资源(灵力)· 一个核心动词(合成)· 一条长期目标(家族越炼越大)。** 桌面上永远只有"一只鸭 + 至多一个会晃的图标"。

---

## 核心循环一图流

```
你写代码(Claude Code / Codex)
        │  Rust 后端 tail 日志,token → 灵力(经真实性节流)
        ▼
[分钟级] 出战鸭播 eat 动画,顶端灵力槽水位上涨 ──── 零操作,纯被动
        │
        ▼
[小时级] 水位每涨够一档 → 解锁 1 次抽签 → 头顶冒"签筒"图标
        │  点一下 = 一次抽签(抽过的档不退还,水位继续涨)
        ▼
[会话级] 抽到一只五行 Gen0 鸭(金/木/水/火/土) → 进"牧场"
        │
        ▼
[博弈时刻] 选两只鸭 → 指定"失败时烧哪只当燃料" → 看成功率% → 点"炼!"
        │  点下的瞬间本地随机数就结算成败(不等 AI)
        ├── 成功 → 走全 AI 融合流水线,几分钟后一只继承双亲的独一无二新鸭"啵"地跳出
        └── 失败 → 你指定的那只 Gen0 燃料鸭"炸炉"消失,退回部分灵力;另一只全保留
        │
        ▼
[长期] 图鉴点亮该配方(格子)+ 家族树多一枝;独一无二实例永久存档、可重铸
        │
        └──────────── 回到写代码,继续攒灵力 ────────────┘
```

**唯一资源 = 灵力。** 抽签、合成、失败返还全部由它派生,不引入任何第二货币。

## 灵力:唯一资源(数据层如实设计)

> **v1 的致命自欺已修正。** 代码事实:`codex_adapter.rs` 里 `project.experience = project.total_tokens / 1000`,它从 `total_tokens` 现算,而 `total_tokens` 单调递增、永不下降(per-session 字节偏移保证重启不双计)。所以 v1"满格清零、继续涨"在数据层根本不成立——清零就得改写持久化的 `total_tokens`,会砸掉"重启不重复计"的核心逻辑。

**正确模型:灵力是一条"水位线",抽签是"解锁刻度",不是"扣款"。**

- `experience`(历史总灵力)**永不修改**,只当水位线读数。
- 新增一个可消费计数器,存进 `collection.json`:`drawn_count`(已抽次数)。
- **可抽次数 = floor(experience / 抽签阈值) − drawn_count**。抽一次 `drawn_count += 1`。
- 玩家侧文案:**"灵力每涨够一档,解锁一次抽签;抽过的档不退还,水位继续往上涨。"** 永不出现"清零"字样,也永不出现"灵力被扣光"的挫败。

这样"唯一资源=现有 experience"的招牌真正成立:全局只读一个水位线 + 一个已抽计数器,零新资源。

### 反刷子:真实性节流(硬需求,非可选)

> token 计数由用户本地明文 JSONL 日志驱动,无校验、无签名。不设防,一个 `while-true` 的 agent 循环或往日志追加假 usage 行,就能无限刷灵力 → 无限抽 → 无限合成,而每次真实 AI 合成都烧真金白银的图像配额。

- **灵力增速硬上限**:单位真实时间内计入的 `token_delta` 有上限(如每分钟最多计 X token),超出部分丢弃或强衰减。异常巨大的单条 delta(远超一次 agent 回合合理值)直接截断。
- **AI 合成配额单独限流**:每日 N 次真实 AI 融合,超出排队或降级为模板皮肤。把"刷灵力"和"烧配额"彻底解耦——即使有人刷爆灵力,也刷不穿你的生成成本。
- 节流参数进"集中可调配置",首版拍脑袋、后续按真实 token 分布校准。

## 五种初始宠物:五行五鸭

以"五行相生相克"作五种初始宠物的骨架,理由:①中国玩家零学习成本、自带相生相克现成规则,省掉一切教学;②五色在 260×320 小窗里一眼可辨;③五行天然闭合——任意两只非同元素鸭必是相生或相克其一,永不出现"这俩能不能合"的困惑。

五只鸭全部延续咕噜鸭"圆滚滚、嘴硬、黏人、爱管闲事、过度自信"的调性,只换配色/小道具/口头禅,**不做任何数值差异**。

| 元素 | 鸭名 | 配色 | 气质 | 口头禅 |
|---|---|---|---|---|
| 金 | 锃锃鸭 | 明黄/金属光泽 + 硬币 | 嘴硬·傲娇 | "我这叫稳,不叫怂。" |
| 木 | 蓬蓬鸭 | 草绿 + 头顶发芽 | 憨慢·过度自信 | "我可是很有生命力的!" |
| 水 | 滑滑鸭 | 青蓝半透明 + 滴水 | 黏人·怕孤单 | "别急,我流着流着就到了。" |
| 火 | 躁躁鸭 | 暖橙 + 尾巴火苗 | 爱管闲事·急性子 | "冲!啊不是那个方向……" |
| 土 | 墩墩鸭 | 赭褐 + 坐得最圆 | 稳重·老好人 | "稳如老墩,你随意。" |

工程落点:`manifest.json` 加 `element`(`metal|wood|water|fire|earth`)与 `generation` 两字段(Rust `avatar_manager.rs` + avatar-gen `types.ts` 双侧同步,serde 已 camelCase),不动状态机。五只鸭复用咕噜鸭 `idle_normal` 帧做配色变体,mock 即可,零 AI 成本。

**用户初始获得一只宠物**:开局固定送 **金·锃锃鸭**(家族原点),其余四只靠抽签,给"凑齐一家五口"留收集空间。

## 喂养:你真实的编程活动(沉浸命脉)

喂养 = 你真实的编程 token,**零额外操作**,是全作沉浸底座。

- **数据源完全复用现有链路**:`codex_adapter.rs` 已把 `token_count` 转 `experience_delta = token_delta/1000` 并持久化 emit;前端已有去重 + 播 eat 动画。**改动只有两处**:①把 experience 面向玩家叫"灵力",累加到顶端一条 6px 细"灵力槽"(实际是水位线);②接上上面的真实性节流。
- **即时反馈三连**:出战鸭张嘴吃(`eat` 帧)→ 头顶飘 `+N` 微字(复用 `showExpPop`)→ 灵力槽平滑上涨。
- **重复鸭回收(牧场溢出的出口,而非收入)**:牧场只 6 格,抽签必出鸭、集齐五行后去重偏向失效 → 中后期多为重复鸭。**牧场满时,新抽到的重复鸭自动折算成少量灵力(不占格、免手动点击)**,消除"反复满、反复手动清"的点击苦力。
  - **死数(平衡命门)**:1 只重复鸭回收灵力 **≪** 抽 1 次消耗的灵力(如抽签阈值 50,重复鸭回收 ≤ 5)。保证"抽→喂掉→再抽"套利期望为负,重复鸭只是"不至于全浪费"的安慰,绝不是收入来源。此兑换率列入"集中可调配置"并标为平衡关键参数。
- **刻意不做**:没有饥饿/心情/清洁度等任何维护性数值,不会饿死,不写代码就正常待机/睡觉(现有 60s→sleeping),**绝不惩罚离线**。喂养只加不减。

## 抽签:喂养的自然溢出物

**通过喂养获得额外抽奖机会,从而获得新宠物。**

- **转化**:灵力水位每涨够一档 → 解锁 1 次抽签 → 桌宠头顶冒会晃的"签筒"图标(**唯一抽奖入口,不占常驻 UI**)。点一下 = 开一次,无货币界面。
- **产出**:抽签 **100% 出一只五行 Gen0 鸭,绝不空抽**。只决定两件事:①元素;②纯度(普通/精纯,精纯仅外观更亮 + 合成时 +5% 成功率)。
- **保底(最简两条)**:
  - 元素:默认五等分各 20%,但**去重偏向**——优先给你没有的元素,直到集齐五种(天然五连保底)。集齐后抽到重复给彩蛋台词("又是你?!"),并按上面的规则自动折算灵力。
  - 纯度:精纯 15%,一条傻瓜保底——**连续 6 次不出精纯,第 7 次必出**。不做卡池 UP / 复杂 pity。

### 首玩放水(修留存)

> v1 的"1 抽 ≈ 5 万 token ≈ 一次会话"过慢:轻度用户可能一整天到不了 5 万,而开局只送 1 只、要凑够 2 只才第一次摸到核心动词"合成"——首玩留存会崩。

- **新账号前 3 抽阈值极低且递增**(如 10 / 20 / 30 灵力),保证**第一天就凑够 2~3 只鸭、当天完成第一次合成**。之后阈值回升到常态(如 50)。
- "首次合成"设为明确的新手引导终点。常态阈值按真实 token 分布(区分轻/重度用户)校准,别用拍脑袋的 5 万一刀切。

## 合成(核心):全 AI 融合 + 五行生克

**两只宠物可合成,走全 AI 合成流程。** 合成是绝对主角,规则 = 五行相生相克,一张表定产物元素(让"发现配方"有逻辑可循)。

### 入口(极小确认框)
牧场选两只鸭 → 弹一个极小"炼丹炉"框,只显示:两个元素图标 + **一个大大的成功率%** + **"失败时烧哪只当燃料"的选择(默认预选代数更低那只)** + 一句嘴硬吐槽。点"炼!"。

### 三档关系 → 成功率 → 产物

| 关系 | 举例 | 基础成功率 | 产物 |
|---|---|---|---|
| 相生(金生水…土生金) | 金+水 | **85%** | 被生元素的升阶鸭。新手最先摸到、最稳。 |
| 相克(金克木…水克火) | 金+木 | **70%**(v1 的 55% 上调) | 克方元素的淬炼变体,反差最大最好玩。中期追求。 |
| 同元素 | 金+金 | **92%** | 同元素高一阶纯度鸭。兜底/刷纯度。 |

> **为何把相克从 55% 上调到 70%**:玩家中后期几乎只做相克与挑战新配方——这正是"核心乐趣"所在。55% 意味着近一半"中期追求"以失败告终,是持续挫败而非偶发敬畏。失败被系统性推到玩家最投入、最不想输的时刻。

### 成功率修正(MVP 极简,只留两项)

```
最终率 = 基础率
       + 每只精纯鸭 +5%
最终率钳制在 30% ~ 95%(永不 100% 保留敬畏;地板从 20% 抬到 30%,配合下方"失败不烧独一无二鸭"托底)
```

> **v1 的加分项在偷偷膨胀成一套数值系统**(图鉴 +10%、代数疲劳 −3%、亲密度加成、残渣兑换……需要持久化 6+ 计数器)。MVP 一律砍掉,只留"基础率 + 精纯 +5%",玩家真正能心算。图鉴 +10%、代数疲劳留到第二迭代再评估。

### 关键:失败结算在点"炼!"的瞬间(不等 AI)

> 全 AI 融合是"数分钟后台异步",若让玩家赌上鸭、等几分钟、才揭晓 55% 没中——漫长等待放大挫败,且玩家会把技术失败(QA 不过/超时)误读成"系统吞了我的鸭"。

- 点"炼!"→ **立刻用本地随机数结算成败,立刻告知**。
- **判定失败 → 根本不调用 AI**(省配额),直接播失败喜剧动画、结算燃料与返还。
- **判定成功 → 才启动融合流水线**,炼丹动画无限循环等待,完成后系统通知(复用现有 emit 通知链路)。

### 全 AI 融合体验(诚实标注:这是接口级重构 + 头号风险)

> **v1 谎称"复用现有流水线、工程量与现有导入路径一致"。代码事实相反:** `ImageProvider.generateIdleAssets(sourcePath: string, ...)` 全链路是**单图**——codex-cli 只传一个 `--image sourcePath`,openRouter 只 `toReferenceDataUrl(sourcePath)` 一张,`gulugulu-avatar-imagegen` skill 的 prompt 写死"from the attached image"(单数)。做双图融合**必须改 provider 接口签名、改 codex args(`--image A --image B`)、改 OpenRouter 的 data-url 组装、重写 skill 让它理解"融合两张"**——这是核心流水线的**接口级重构**,不是扩展。四处要动:`imageProvider.ts` / `codexOrchestrator.ts` / `openRouterProvider.ts` / imagegen skill。

- 成功判定后,新增 `generateFusionAssets(specA, specB, targetElement)`:合并两份 `character-spec.json`(各取标志性 `visualTraits[]` + 五行混色 + 拼新 `name`/`summary`),把双亲 `source.png` / `standard-design.png` 作**双参考图** + 目标元素提示喂给 provider 生成融合设计稿;之后复用 pipeline 后半段(切帧 → 抠图 → QA → WebP → manifest 带 `element`+`generation` → zip)。
- 完成 → 桌面 App `install_avatar_from_url` 装上,新鸭从光里"啵"地跳出(`idle` 首帧淡入),飘出继承双亲的口癖混搭台词。
- **合成期间桌宠钻炉,你继续写代码攒灵力**,炼好冒泡通知——这是硬需求,不是彩蛋。

**耗时如实标注**:单图 codex job 超时上限已设到 **900,000ms(15 分钟)**;融合更重(双图 + 更复杂 prompt + 16 帧姿态表 + 切帧抠图 + QA)。**真实融合 P95 待压测,大概率数分钟级**。v1 的"几十秒"是 mock 速度,已删除。

**独一无二的兑现范围(不 over-sell)**:融合**只定制 `idle_normal`**,其余状态回退内置咕噜鸭帧。文案如实表达为"换了待机形象的独一无二皮肤",不宣称"每一个动作都是新鸭"。若第二迭代预算允许,可把 `happy_dance`(合成庆祝)也纳入融合(2 个状态),但那会翻倍 AI 成本,需单独评估。

## 失败机制:有代价但不劝退(重做)

**合成有一定失败概率。** 核心手法两条:**失败率永远看得见** + **失败只烧"可再生材料",绝不摧毁不可再生的记忆**。

### 红线一:独一无二的鸭永不被烧

> v1 最致命的自相矛盾:一边花全作最大力气让玩家爱上"用一周 Rust token 炼出的锃金涌泉鸭",一边让它有 15%~45% 概率被炸炉永久消失。灵力可再生,那只鸭不可再生——残渣返还灵力对"刚被烧掉的唯一形象"毫无安慰。这是玩家愤而卸载的头号触发点。

- **真正会"炸炉"消失的,只能是可无限抽签补充的 Gen0 五行普通鸭。**
- **Gen1+ / 合成产物 / 精纯鸭永不被销毁**:合成失败时它们**原地全保留**,只消耗那只 Gen0 燃料。
- 因此"用珍爱的鸭去合成"最坏结果 = 亏掉一只随时能再抽的 Gen0 材料,而非失去那只招牌鸭。

### 红线二:牺牲品由玩家指定,不由系统随机

> v1 的"随机 / 较低亲密度那只被消耗"会精确烧掉玩家最爱的。系统替玩家选牺牲品 = 玩家觉得"被系统背刺",而非"自己赌输"。同一份损失,归因不同,情绪天差地别。

- 炼丹炉里点"炼!"前,玩家**明确指定"若失败烧哪只当燃料"**(默认预选代数更低那只)。玩家自己钉的牺牲品,输了是"我认赌"。
- 若两只都是 Gen1+(都不可烧),则该次合成**失败也不销毁任何鸭**,只退还部分灵力——鼓励玩家用"珍藏 × Gen0 材料"的组合去博。

### 失败后果 = 纯灵力返还(删掉残渣第二货币)

> v1 的"残渣可当灵力 / 攒 3 坨兑 1 抽"是事实上的第二货币(独立库存、独立兑换率),既破"只有一种资源"红线,又给刷子第二条"无脑合成→失败吃残渣→换更多抽"的自循环管道。

- **失败直接结算成灵力返还**(如返还所消耗 Gen0 鸭价值的一部分灵力),**不新增任何库存物、不发抽签机会**。守住纯减法、唯一资源。
- "它把力量留给了你"的情绪,用返还灵力时的气泡台词表达即可,不需要一个新道具。

### 情绪缓冲:演出随损失体量分级

> 让招牌鸭"顶着焦毛嘴硬:哼我故意的"是在玩家真实肉疼的伤口上抖机灵,会被读成系统嘲讽。语气必须匹配损失分量。

- **Gen0 普通鸭炸炉**(唯一真正会消失的情况):走全套滑稽喜剧——焦毛、黑脸、"哼,我故意的,这次不算!"。损失轻,喜剧得体。
- **Gen1+ 只是"合成没成功、但没死"**:走轻微遗憾但向前的台词("差一口气,再攒攒灵力就来"),不抖机灵、不说俏皮话。

### 结构性防挫败
- 相生 85% + 相克上调到 70%,新手前几炼大概率成功,先尝甜头。
- 失败只烧 Gen0(可无限补充),挫败被封顶在"亏一只随手材料"。
- 30% 地板价兜底;技术性失败(QA 不过 / 生成超时)**免费重跑,不算游戏失败、不扣任何鸭**,且有明确不同的文案与视觉("炉子卡住了,鸭没事,重来"),与游戏失败彻底区分。
- **不做保护道具 / 复活币 / 氪金重炼**,避免把一次豪赌变成经济系统。

## 成长与图鉴:配方 × 实例(切分清楚)

> v1 的"完全可推导"与"独一无二"打架:同一配方(金+水)每次 AI 生成都不同,那"图鉴点亮一格"到底点的是格子还是那只具体的鸭?两个概念必须显式分开。

- **图鉴格子 = 配方(元素组合)**:可推导、可集齐、点亮一次。金+水 这个格子第一次炼成就点亮,记录"你解锁了这个配方"。
- **具体某只鸭 = AI 生成的实例**:独一无二,同一配方可拥有多只不同实例。
- **家族树**:记录你抽到过、合成过、失去过的每只鸭实例,合成体显示双亲连线。

### 永久存档 + 重铸(解除"雪藏焦虑")

> 若产物鸭进牧场后玩家因"怕它当材料被消耗"而不敢再用,合成循环就停摆。而事实上 Gen1+ 已经"永不被烧"了,但仍需一个正向语义。

- **任何鸭一旦抽到/合成,其形象+名字+台词永久进图鉴存档。** 即使某只 Gen0 实例被当燃料消耗,其"配方形象"也在图鉴里。
- **重铸**:图鉴里已存档的形象可花灵力复刻一只回来。于是"用鸭去合成"= "它去了但随时能召回",爽感与循环双保。

### 长期内容:开放指标,不设死上限

> v1 用"图鉴 20~30 格上限 + Gen2 封顶"把内容锁死,一个中度用户几天点满就进入"抽重复→喂掉"的纯数字增长——撑不起"随编程月份慢慢长大"的承诺。既然每只合成鸭都是 AI 生成的无限不重复实例,就别既承诺长期又设死上限。

- **定位取舍(二选一,文档明确选后者)**:让长期成长指标改为**开放式**——"你炼出过多少只独一无二的鸭 / 家族树长到第几代 / 集齐了多少种配方组合",而非可数的 25 格通关条。
- **代数**:每只鸭有 `generation`(初始=0,产物=max(双亲)+1)。代数越高外观越华丽、口头禅越中二,是血统炫耀。**MVP 不设代数疲劳、不设 Gen 封顶**;总 AI 调用量由上面的"每日 N 次配额限流"控制,而不是靠封顶——这样既控成本又不砸掉长期承诺。
- **稀有度**:普通 / 精纯两档(精纯 = 同帧 + CSS 描边发光滤镜,零额外美术)。

## 沉浸感设计(别人抄不走)

1. **写代码 = 炼灵力,可见**:桌宠浮在 IDE 旁,你每写一段、AI 每花一批 token,它当场咬一口、灵力槽当场涨。工作时角落有只鸭陪你、靠你干活长大。
2. **"你写的代码有味道"(招牌隐线,第二迭代)**:后端按会话 token 来源(项目/文件类型/时段)给灵力打隐性元素标签,让喂养气泡吐槽"这行代码有股金属味",并让对应元素抽签概率轻微加权。把真实编程活动翻译成有性格的灵力。非 MVP(需在 codex_adapter 事件加 `element_hint` 字段)。
3. **活动共情**:宠物已能随 agent thinking/working/success/error 切表情——debug 报错多时墩墩鸭安静蹲你旁边;连续 success 时锃锃鸭嘴硬"还行吧(其实很得意)"。
4. **"记得你"(第二迭代)**:利用已持久化的 per-project experience + 新增"上次陪伴时间戳",久别回来说"你三天没来了";深夜写码触发"陪你熬夜"低语。
5. **"独一无二"的兑现**:全 AI 融合让每只产物形象/名字/口头禅现场生成——世界上没有第二只一样。截图炫耀是最强自发传播点(但如实标注只有待机态是新皮)。
6. **尺寸克制**:见下节。

## UI:桌宠窗口只留最小交互,大网格移到网页

> v1 把牧场 6 格 + 图鉴 20~30 格 + 炼丹炉 + 成功率明细 + 家族树全塞右键菜单。现有机制已在硬撑——context menu 显示 avatar 列表时已必须调 `getCurrentWindow().setSize()` 撑大原生窗口才不被裁。再塞这些滚动大网格,小窗会被撑成半屏弹窗,"极简桌宠"荡然无存。

- **桌宠窗口只保留三样**:①顶端 6px 灵力槽;②满档冒出的签筒图标;③炼丹炉——"两个元素图标 + 成功率 + 燃料选择 + 炼!"这一个极小确认框。牧场(6 格,可放右键菜单二级项,勉强不撑窗)。
- **图鉴 / 家族树这类需要滚动大网格的界面,不塞进桌宠窗口**。复用现有 `open_avatar_generator` 那条"开外部窗口/浏览器"的路子,把图鉴和家族树做成 avatar-gen 服务里的一个网页页面。桌面永远只有"一只鸭 + 至多一个会晃的图标"。

## 刻意保持简单(不做什么)

- **只有一种资源**:灵力(= experience 水位线 + drawn_count 计数器)。砍掉金币/钻石/体力/材料/**残渣**/一切第二货币。
- **只有一个核心动词**:合成。不做战斗、伤害计算、PVP、关卡、任务、剧情。五行只影响合成产物元素倾向,不参与任何对抗。
- **不做维护性数值**:没有饥饿/心情/清洁/亲密度面板,不写代码 = 鸭子睡觉,零损失,永不惩罚离线。
- **不做等级/属性点/升星**:成长 = 家族树多一枝 + 解锁新配方。
- **成功率 MVP 只两项(基础率 + 精纯 +5%),钳 30~95%,可心算。** 图鉴 +10%、代数疲劳、亲密度加成全砍或延后。
- **保底只留两条**:抽签必出鸭、6 次不精纯第 7 次必出。
- **合成只定制 `idle_normal`**,其余回退内置帧,如实标注不 over-sell。
- **存储只加一个 `collection.json`**(拥有列表 + 图鉴/配方解锁 + `drawn_count` + 元素/纯度保底计数器),紧挨现有 `avatar-config.json`;manifest 只加 `element` + `generation`。
- **数值参数集中在一个可调配置**,首版拍脑袋、后续调参。

## 招牌点

1. **五行相生相克驱动合成**:产物元素与成功率可推导,"发现配方"有逻辑;文化共鸣 + 零教学。
2. **"你写的代码有味道"**:真实编程活动被翻译成有元素性格的灵力,塑造你专属鸭群——换皮桌宠抄不走。
3. **失败 = 亏一只随手材料 + 力量返还,珍藏的鸭永不消失、还能重铸**:情绪落点是"再攒攒就来",而非"回档"的愤怒。

## 首个可玩版本(MVP)范围

目标:用最小改动跑通"喂养→抽签→合成(可失败)→图鉴"闭环。

### MVP 第 0 步(前置 spike,先做,决定生死)
> **把"codex-cli / openRouter 双参考图能不能融出一只连贯、延续调性的 idle 鸭"当成 MVP 头号前置 spike。** mock provider 根本不读输入图(`_sourcePath` 是丢弃参数,只 hash `spec.name` 画通用 SVG),用 mock 跑通只证明"拼两个 name 能出图",对真正的图像融合零证据。
- 先做 provider 接口双图改造(4 处),跑 **≥20 次真实融合**,量成功率 / P95 耗时 / 单次配额消耗。**这决定整套玩法是否成立**。达标才继续;不达标则回退方案(见下)。

### MVP 做
1. 灵力槽 UI(水位线 + 满档签筒)+ 真实性节流。
2. `collection.json` + Rust 命令 `draw_pet` / `list_pets` + 重复鸭自动折算灵力。五行五鸭换皮(idle_normal 配色变体,mock)。开局送金·锃锃鸭。首玩前 3 抽放水阈值。
3. 抽签:100% 出鸭 + 去重偏向 + 精纯 6→7 保底 + 牧场 6 格。
4. 合成:炼丹炉确认框(成功率 + **玩家指定燃料**)+ 三档基础率 + 精纯 +5% + **点"炼!"瞬间本地结算** + 失败只烧 Gen0 + 纯灵力返还。
5. 融合流水线:`generateFusionAssets(specA, specB, targetElement)` 双图版。**mock 只验证 UI/资源循环,codex-cli/openRouter 才是兑现点**。异步 + 钻炉动画(无限循环等待)+ 完成系统通知。
6. manifest 加 `element` + `generation`;图鉴最小版做成 avatar-gen 网页(配方格子 + Gen1 剪影)。

### MVP 兜底(spike 若不达标)
QA 不过 / 超时 / 配额耗尽 → 免费重跑 → 仍不行则回退"五行混色模板皮肤(普通稀有度)"保底出货,**绝不给空结果**。若 spike 显示真实融合根本不可用,则玩法降级为"模板混色皮 + 名字/台词 AI 生成",诚实调整"独一无二"叙事。

### MVP 暂不做
家族树可视化、"你写的代码有味道"元素加权、"记得你"陪伴台词、代数疲劳、图鉴 +10%、重铸、happy_dance 融合。

### 决策 1:真实性节流的灵力增速硬上限设多少?
- **A.** 每分钟计入上限 = "一次典型 agent 回合的合理 token 值"(需先采样真实分布)
- **B.** 按会话总量封顶(单次会话最多贡献 X 灵力)
- **C.** 不设死上限,只截断异常巨大的单条 delta
- **建议**:先采样真实 Claude/Codex 会话 token 分布,把"每分钟上限"定在 P95 正常回合略上方——不误伤重度真实用户,又拦住 while-true 刷子;上线前用真实数据校准,别拍脑袋。

### 决策 2:两只都是 Gen1+ 珍藏鸭合成失败时,是否需要额外代价避免"零风险刷配方"?
- **A.** 失败零实体损失,只是本次没出新鸭(纯时间/灵力成本)
- **B.** 失败时其中一只降一档纯度(精纯→普通)作为温和代价
- **C.** 禁止两只 Gen1+ 直接合成,强制至少一只 Gen0 材料
- **建议**:选 A——珍藏鸭本就稀缺,"炼失败=白等一场+灵力沉没"已是足够代价;强加降纯度会重新引入"烧掉珍视资产"的卸载风险,与本次核心修正背道而驰。

### 决策 3:若 MVP 第 0 步 spike 显示真实双图融合成本/质量不达标,玩法降级到什么程度?
- **A.** 完全降级:模板混色皮 + AI 只生成名字/台词,放弃"图像融合"叙事
- **B.** 部分降级:单图融合(只取一只当主形象)+ 另一只贡献配色/道具
- **C.** 暂缓合成玩法上线,先只做"抽签 + 收集五行"的轻量版
- **建议**:优先 B(单图主形象 + 另一只贡献配色)——复用现有单图链路、成本可控,又保留"两只合成出新鸭"的核心动词;把"全双图融合"作为达标后的增强项,而非上线阻塞项。
