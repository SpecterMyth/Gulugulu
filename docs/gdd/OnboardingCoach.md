## 1. 设计目标与原则

1. **"现在做什么"零歧义**：任一时刻只有**一个**高亮目标 + 一个手势动画，直接指向要点击的物体或要按的键。不靠玩家读长文。
2. **手势即语言**：核心是两种动画——**手指点击**（tap，点物体/按钮）与**按键图标**（keycap，教键盘操作：左右移动、敲键盘回精力）。文字只做 ≤12 字补充标签，且**复用现有对话气泡组件渲染，不新增任何文本组件**（见 §5）。
3. **线性但不阻断**：教练按固定顺序推进，但**从不禁用**任何操作。玩家乱序/提前完成 → 该步自动跳过并重锚到下一步；卡住 → 手势耐心停留。永远正向、无报错、无惩罚。
4. **强度有始有终**：这是**首跑专属**的重引导。**首次融合一完成立即整层退场**，之后交给轻引导气泡（[OnboardingGuidance.md](OnboardingGuidance.md)）——避免"老手仍被戳手指"的烦躁。
5. **表现克制**：纯 SVG/CSS 动画、data URI、无外部资源（满足 CSP `img-src self/data:`）；只用 `transform/opacity`；`prefers-reduced-motion` 降级为静态箭头+文字；`document.hidden` 暂停。

## 2. 手势动画词汇（可复用原语 · CoachFx）

一套与物种美术同源的**代码绘制**原语，锚定到目标屏幕坐标（§6.2 锚定机制）：

| # | 原语 | 视觉 | 用在哪 |
|---|---|---|---|
| **G1 手指点击** | 👆 手指从斜下探入 → 轻点 → 落点扩一圈波纹 → 回弹，~1.1s 循环 | 收蛋、点击打工、菜单 🏡、商店蛋卡、陪伴/融合按钮 |
| **G2 手指连点** | G1 加速版（~0.5s/次）+ 小连击星，暗示"多点几下" | 连续打工、点到满级 |
| **G3 按键图标** | 圆角**键帽** SVG（复用 voltmouse 键帽原语）浮起脉冲 + "按一下"微光；成对显示 `◀ ▶` | 后院左右移动（`← →` / `A D`）、敲键盘回精力（`⌨` 任意键） |
| **G4 边缘方向箭头** | 屏幕边缘朝目标方向的脉冲大箭头（目标在视口外时） | 走去商店 / 走向另一只宠物（POI/角色在屏幕外） |
| **G5 目标高亮环** | 目标周围一圈呼吸描边光环（**只描边、不压暗全屏**） | 所有步骤的"就是这个"强调，配合 G1~G4 |

- **手指素材（已定）**：**自绘矢量小手**（与 kawaii 风统一），只需 **tap / 连点**两姿（无拖拽——移动走按键、融合走按钮，全程无拖拽手势）。放 `coach/CoachHand.tsx`。
- **键帽素材**：复用 `voltmouse` 键帽/`FlightLayer` 键帽雨的现成矢量件，风格一致。
- **合成**：`CoachFx` 只画 **1 组手势 + 1 个高亮环**（**不含任何文字**——引导文字与进度全部走现有对话气泡，见 §5），绝不叠多个焦点。

## 3. 教练步骤序列（状态机）

线性 7 步 + 1 个事件插播（精力恢复）。下表给每步的**终点动作**（触发前置 → 指向目标 → 手势 → ≤12 字标签 → 完成条件），完成即推进、**已满足则自动跳过**（乱序/老玩家自愈）。因引导**非阻塞**、玩家可能在任意界面——**如何从当前上下文走到该终点动作**的分支路由见 **§3.2**。

| 步 | 目标动作 | 触发前置 | 指向目标 | 手势 | 标签文案 | 完成条件 |
|---|---|---|---|---|---|---|
| **C1** | 孵第一个蛋 | 无精灵、有孵化中的教学蛋 | 发光的蛋（舞台/后院孵化槽） | G1 + G5 | 蛋好了，点它！ | `collectHatched` 成功、`pets≥1` |
| **C2** | 连续点击打工 | 有主宠、`uiMode∈{pet,menu}` | 主宠身体 | **G2**（收益飘字已有） | 快连点我打工！ | 累计点击 ≥ `COACH_WORK_CLICKS`(默认 12) |
| **CE** | 敲键盘回精力（插播） | **任意时刻主宠首次 `exhausted`** | 主宠 + 键盘区 | **G3(⌨)** + G5 | 敲键盘喂我回精力⚡ | 键盘充能入账 ≥1 或精力≥唤醒阈值 |
| **C3** | 进后院 + 左右移动 | C2 完成 | 菜单 🏡 → 后院角色 | G1(点🏡) → **G3(`◀ ▶`)** | 去后院走走 / 按左右移动 | 进入 `backyard` 且 `charX` 双向移动过阈值 |
| **C4** | 去商店买第二颗蛋 | C3 完成、精灵数<2 | 商店 POI（屏外→G4；近→G1 蛋卡） | **G4** → G1 | 走去商店买颗蛋 | `buyEgg` 成功（第二颗蛋入手/入孵） |
| **C5** | 切换陪伴宠物 | 两只精灵都在后院 | 非主宠脚下"陪伴"按钮 | G1 + G5 | 走近它，点"陪伴" | `setActivePet`(onFollow) 切到另一只 |
| **C6** | 两只点到满级 | 拥有两只同阶精灵 | 当前未满级的精灵 | **G2** | 点满它！还差 {n} 下 | `fusionReady` = 两只同阶满级凑齐 |
| **C7** | 走近另一只 → 融合 | `fusionReady` | 另一只满级精灵（屏外→G4）→ 脚下"✨融合"大牌 | **G4** → **G1** | 走过去，点融合✨ | `fusePets` 成功 → **整层退场** |

**要点**：
- **CE 精力恢复**是**事件插播**（不占固定顺位）：主宠首次打空一管（`clickWork` 返回 `becameExhausted`）时插入，教一次键盘回精力（G3 `⌨`），学会即销号；之后再耗尽只由轻引导气泡兜底。

> ⚠️ **待决**：曲线重平衡后**一阶满级只需 45 击，而一管精力 = 100 击**（`staminaMax 200 ÷ staminaPerClick 2`），原先"点满第一只必然打空一管（~225 击 > 200）"的前提**已失效**——CE 的触发条件（`clickWork` 返回 `becameExhausted`）在 C2/C6 阶段不会再自然触发，教学期玩家可能一次都碰不到精力耗尽，这一课将永远教不出去。**需重新设计触发时机**（候选：改由二阶及以上养成期触发、教学期临时下调精力上限、或改为"首次跌破某个低精力阈值"即插播）。本次只标记断裂，不改动触发机制的设计。

- **C2 只教"手感"**（点 ~12 下尝鲜），**C6 才是真正练满**（两只各 45 击，各自半管精力内即可完成）。分开避免 C2 就变成漫长苦练。
- **C4/C7 的"走过去"**：POI/另一只宠物常在视口外——先 G4 边缘箭头指方向（配合 G3 左右键"往这边走"），走近后 POI 弹板/动作牌出现，再 G1 点具体按钮。
- **C6 的进度**：气泡 label 实时读 `expToMax` 折算的"还差 {n} 下"（复用 `tutorial.ts` 现成算法，即 ambient `expdiff` 同款文案），把满级里程碑"留在玩家手指上"——**进度也走气泡，不另立组件**。
- **融合前置未满足**时（如金币不足/未满级），C7 手势暂不催"点融合"，改指向缺口（继续点满 / 攒钱），复用 `fusionHintFor` 的阻断原因文案。

### 3.1 教学期孵化硬编码（让 C1/C4 秒级闭环）

教学期把两颗关键蛋的孵化时间**写死**，"孵蛋→收蛋"秒级闭环，玩家不必干等 3–8 分钟：

| 蛋 | 正常 | 教学硬编码 | 落点 |
|---|---|---|---|
| **开局教学蛋**（C1 收的那颗） | 60s | **15s** | `config.hatchSeconds.tutorial` 60→**15**（`config.json` + `config.test.json`）。Rust `persist.rs:11` 与 mock `mockEngine.ts:191` 都读它——**纯配置值，无代码改动**。 |
| **首次商店购买蛋**（C4 买的那颗） | 一般 180s / 火水草 300s / 电冰 480s | **固定 30s**（仅第一次购买） | `buy_egg` 判"首次购买"→ `hatchAt = now + 30`。需一枚存档标记 `tutorialFirstEggBought`（serde default false，首购后置 true）。Rust `logic/economy.rs:120` + mock `mockEngine.ts:457` 各加一分支。 |

- **只影响头两颗**：教学蛋恒 15s；**第一次**购买蛋恒 30s，之后购买按正常时长——不破坏正式经济。
- **与 coach 解耦**：即使玩家跳过教练，这两条照常生效（首购 30s 由存档标记判定，不依赖 coach 激活）——是"游戏前两颗蛋"的硬规则，不属动画层。
- **这是本方案唯一的后端触点**：一处 config 值（15s）+ 一处 `buy_egg` 首购分支 + 一枚存档 bool（30s）。教练动画/文字层本身仍纯前端。

### 3.2 非阻塞全界面路由（每步 = 目标 + 上下文分支子序列）

引导**非阻塞**：任一时刻玩家可能处于任意 `uiMode`（pet / menu / backyard / settings / debug）、主宠是任意一只、目标宠可能在屏外。所以**每一步不是"指一个固定目标"，而是一个终点动作 + 一段"从当前上下文走到它"的子序列**。

**解析模型（resolver）**：把每步的终点动作拆成一串**前置阶梯**（有序 preconditions）；`useOnboardingCoach` 每次求值**找第一个未满足的前置**，显示它对应的**导航子步**（复用下方原语的手势 + 现有气泡 label）。玩家不管在哪、做过什么，教练都能重新解析出"现在该点哪 / 按哪键"。全部满足 → 终点动作的完成事件推进下一步。**所有子步只用 §2 的手势 + 现有气泡，零新组件。**

**导航原语（子步，跨步复用）**：

| 原语 | 触发（第一个未满足的前置） | 指向 | 手势 | 气泡 label | 满足 |
|---|---|---|---|---|---|
| **N0 退出面板** | 在 settings/debug 而目标不在此 | 返回处/空白 | G1 | 先返回 | uiMode→pet/menu |
| **N1 展开菜单** | `uiMode==pet` 而需菜单/后院 | 主宠身体 | G1 | 点我展开菜单 | `uiMode==menu` |
| **N2 进后院** | `uiMode==menu` 而目标在后院 | 🏡 菜单项 | G1 | 进后院 | `uiMode==backyard` |
| **N3 走向目标** | 在后院、角色远离目标宠/POI | 屏外→边缘箭头 | G4 + G3(←→) | 往这边走 | 角色进入目标 range |
| **N4 攒够金币** | 终点动作需金币而 coins 不足 | 就近可点的宠（`pet` 态先 N1） | G2 | 再点几下攒金币 | `coins ≥ 所需` |
| **N5 先收待收蛋** | 有已孵化未收的蛋挡住流程 | 发光蛋/孵化槽 | G1 | 先收下它 | `collectHatched` |
| **N6 切换陪伴** | 终点需要某宠成为 `activePet` | 该宠脚下"陪伴"牌 | G1 | 点陪伴换它跟随 | `setActivePet` |

> N3 目标坐标读 `BackyardScene` 的 `placedPets[].x`（宠）或 `SHOP_CENTER_X` 等 POI 常量；N1/N2 复用现有 uiMode 切换；N4 内部复用 N1（`pet` 态先展开菜单再连点）。

**每步的前置阶梯**（从上到下取第一个未满足项显示；全满足 = 直接终点动作）：

| 步 | 前置阶梯（依次） | 终点动作 |
|---|---|---|
| **C1 收蛋** | N0 | 点发光蛋（**舞台 `stageEgg` 或后院孵化槽就近点**，不强制进后院）；未到点气泡报"还有 {s}s" |
| **C2 学点击** | N0 → N1(若 pet) | G2 连点主宠（**menu 态主宠 / 后院角色皆计数**） |
| **CE 回精力**（插播） | —（键盘全局，任意界面生效；主宠不可见时居中显示 ⌨） | 就地 G3(⌨) 指当前主宠表示，敲键盘回精力 |
| **C3 进后院+移动** | N0 → N1(若 pet) → N2(若 menu) | 在后院 G3(←→) 左右各移动过阈值 |
| **C4 买第二颗蛋** | N4(coins<最便宜蛋) → N0 → N1 → N2 → N3(走向商店) | 商店弹板里 G1 点蛋卡（`buyEgg`） |
| **C5 切陪伴** | N5(2 号蛋已孵未收→先收；仍孵化中→气泡"等蛋孵好") → N0 → N1 → N2 → N3(走向另一只) | G1 点"陪伴"（切到另一只） |
| **C6 点满两只** | **见下分支** | G2 连点未满宠，"点满它！还差 {n} 下" |
| **C7 融合** | N0 → N1 → N2 → N3(走向另一只满级宠) → N4(coins<融合费) | 其脚下 G1 点"✨融合"（`fusePets`）→ **coachDone 退场** |

**C6 分支**（目标 = 任一未满级宠；两只都未满时优先"就近可达"的那只，主宠优先省一次进后院）：
- **未满宠 = 当前主宠** → 就近点：`pet 态先 N1 展开菜单`，之后 `menu / 后院` 直接 G2 连点。
- **未满宠 ≠ 主宠** → 必须去后院点它：`N0 → N1(若 pet) → N2(若 menu) → N3(走向它) → G2 连点`。

> **对照用户示例**（C6、当前主宠已满、另一只未满、在主界面 `pet` 态）→ 命中"未满宠 ≠ 主宠"分支，逐步解析为：
> **① N1 点主宠展开菜单 → ② N2 点 🏡 进后院 → ③ N3 左右移动找到另一只 → ④ G2 连点它至满级** —— 与需求一字不差。玩家中途若切回主界面、或先去点了主宠，resolver 下一帧重算，指引自动跟上。

## 4. 生命周期与持久化

### 4.1 教练是否激活

```
coachActive = !coachDone && !everFused
```

- **`everFused`（存档派生，权威兜底）**：`save.pets 或 save.eggs 有 tier≥2` **或** `dexObtained` 含任一多元素物种。老玩家/新设备空 localStorage 也不会被重新教。
- **`coachDone`（localStorage 闩）**：C7 的 `fusePets` 首次成功时置位，立即整层退场。
- 二者取或：任一为真即结束。首跑玩家两者初始为假 → 教练开跑。

### 4.2 当前步推导

- **步指针** `gulugulu.coachStep`（localStorage）+ **每步"已满足"自愈**：进入某步前先按存档判断该步是否已达成（如已有 2 只精灵 → 跳过 C4），达成则直接前移，保证乱序/中途安装也从正确位置继续。
- 纯存档可判定的步（C1 有宠、C4 精灵数、C6 fusionReady、C7 everFused）以存档为准；**仅表现类**的步（C3 是否移动过、C5 是否切换过）用 localStorage 完成标记补足。
- **步内子步**（"此刻该点哪"）不持久化：由 §3.2 的前置阶梯 resolver 按当前 `uiMode`/位置/主宠实时求值，玩家一切换界面即重算。

### 4.3 退场与重置

- **退场**：`coachDone` 置位后，`CoachFx` 层不再挂载；轻引导气泡（[OnboardingGuidance.md](OnboardingGuidance.md)）接管。首次融合产出的是 **2 阶蛋**（30 分钟孵化）→ 2 阶宠孵化时触发 `graduation` 气泡，衔接顺滑。
- **跳过（已定）**：教练**首帧**（C1）角落给一个小"跳过教程"入口，点击 → 置 `coachDone`、整层永久退场（急性子/老玩家一键关掉）。
- **不可重看（已定）**：**不提供**"重看教程"入口——`coachDone` 一旦置位（融合完成或主动跳过）即终身不再出现。

## 5. 与轻引导（气泡层）的分层：复用同一气泡

**教练的引导文字直接复用现有对话气泡组件，不新增任何文本组件**（用户要求）。手势层与文字层是一套演出的两半：`CoachFx`（手指/键帽/箭头/环，**无字**）指向目标，**现有气泡**显示 label——二者**同屏协作**（不是互斥）。

- **文字走现有气泡多路复用**：`useOnboardingCoach` 输出当前步 `label`（≤12 字），插入既有气泡的优先级链：
  - 主舞台（C1/C2/CE）→ `App.tsx` 的 `SpeechBubble`：**状态 toast > 教练 label > ambient hint > 随机台词**。
  - 后院（C3–C7）→ `useCharSpeech` 的 `charSay`（同一角色气泡），同优先级。
- **ambient hint 让位**：`coachActive` 时 [OnboardingGuidance.md](OnboardingGuidance.md) 的 `computeTutorialHint` 被 `useTutorialHints(suppressed)` 抑制返回 null——不是两个气泡叠加，而是**同一气泡改由教练驱动**。
- **toast / 报错照常**（即时反馈优先级最高，转瞬即逝，之后 label 复位）。
- 退场（首次融合）后气泡交回 budget 系统；**毕业气泡**（首只 2 阶孵化）正好承接。

## 7. 边界与非目标

- **不阻断、不强制键盘**：C3/C4 的移动，键盘（G3）是"更快"的建议；`walkToPointer`（点地面走过去）永远兜底，手指点地面也能走。
- **不做拖拽融合**：现行融合是"走近 → 点✨融合按钮"（`BackyardNearPetActions`），教练据此设计（G4 走近 + G1 点按钮），**不引入**长按拖拽输入。
- **平台**：键帽图标面向桌面键盘；无键盘环境（纯触摸）退化为 G1 点地面移动 + G4 箭头。
- **性能红线**：同 [InteractionEconomy.md](InteractionEconomy.md) §6.7——仅 `transform/opacity`，`document.hidden` 停。
- **i18n**：标签 ≤12 字，走 i18n key（补 EN；现状轻引导为 zh-only，一并规划）。
