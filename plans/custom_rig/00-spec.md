# AI 融合"专属 rig"设计准则（Custom Rig Spec）

> 创建 2026-07-15。方向由用户拍板：AI 融合物种走**全 AI 手绘成数据的三视图专属 rig**
> （不是参数化 chimera 拼装）。本文件是这条方向的**唯一事实源**——Phase 3 的生成
> prompt 必须逐条实现下面的准则。概念样板见 `projects/gulugulu-app/scripts/render_customrig.tsx`
> 的手搓「晶甲龟」。

## 一、为什么是"数据驱动的 rig"，不是代码

固定配方物种是**编译进 app 的 `.tsx` 组件**（一物种一文件）。AI 融合物种在**用户机器上运行时**
现生，app 已打包、没有编译器，塞不进新代码 → 融合物种只能表达成**纯数据**（`ShapeNode`
白名单，被通用解释器画出来）。所以"手绘专属 rig" = **让 AI 把每个部件的形状当数据输出**。

## 二、动画白拿的硬前提（不可违反）

14 态动画（呼吸/迈步/摆尾/挥手/庆祝/进食/睡眠…）是**免费**的，**当且仅当**身体被拆进标准
`<Part name="body/head/armL/armR/legL/legR/tail/headtop/tool">` 并给对 pivot。因此：

- AI 只画"**每个部件在自己局部坐标系里的形状**"（(0,0)=该部件锚点）；**渲染器负责摆放 +
  pivot + 包 Part**（`customSpecies.CustomDataRig`）。AI 不碰 pivot、不碰 Part 结构。
- 数据模型：`CustomRig = { front, side?, lie?, floating? }`，每个视图是 `RigViewParts`
  （body/head/face/belly/armL/R/legL/R/tail/headTop/decor + 各锚点）。镜像 `rigTypes.ts` / `types.ts` / `game.rs`。

## 三、设计准则（Phase 3 prompt 逐条落地）

1. **可爱幼态**：头大、占比高（头形随原型走：喙/长吻/扁嘴/菌帽皆可，不必正圆），脸饱满、眼睛清晰可读；整体圆润胖乎的幼态比例。
2. **剪影差异最大化**：和固有 6 底座（鸭/狐/鼠/鲸/菇/雪怪）**形体差异尽可能大**；
   融合体要一眼看出是"新物种"（龟壳/长颈/多足/带翼/球形……皆可）。不同融合体之间也要各异。
   - **落地机制（关键）**：`build_prompt` 按 `egg_id` 哈希从 `BODY_ARCHETYPES`（12 种：四足/长颈/带壳/飞鸟/蛇形/多足/高瘦/宽扁/长吻/漂浮/双足猛兽/大附肢）**轮转注入一条"本次体型"硬指令 + 构造搭法**，逼每颗蛋走不同体型、禁止回到"圆头+圆身"。否则模型会被示例锚定、总画圆球。多足把最前一对腿放 legL/legR、其余画进 body；漂浮设 `floating:true`。
   - **三种构造模式（schema 层放开，2026-07-16）**：①分体式（head+body 各画各的）；②**一体式**（整只一个大块画进 body，`head:[]`，headX/headY 指到体块上的脸位——海象锥体/灯笼鱼球）；③**头即全身**（主形画进 head，`body:[]` 或只画细梗——向日葵盘/气球怪）。校验只要求 body/head **不同时为空**；脸的位置永远由 headX/headY 决定，与 head 有无形状无关。这是打破"两球默认形"的结构性前提。
3. **真三视图**：
   - `front` 正面站立（主视图）。
   - `side` **真侧视、朝右**（moving 用）；左移由外层 CSS `scaleX(-1)` 翻转。**禁止**用正面平移冒充移动。
   - `lie` **合理的趴卧睡姿**（专门画的蜷卧/趴地构图，头搁身前之类）；**禁止**只把站姿压扁。
4. **每个动作不同的夸张表情**：表情走表情库随 `petState` 切换，要**夸张、可读、各不相同**
   （待机眨眼 / 移动开心 / 打工用力 >< / 庆祝星星眼 / 进食咀嚼 / 思考上瞟 / 出错蚊香眼+汗 / 睡觉闭眼）。
5. **专属打工工具**：每个物种一件**契合主题**的工具（`toolId`，工具目录见 `fusionParts.json`），
   working/庆祝时握持出现。
6. **打工粒子 = 工具的真实产物**（★本轮新增，最重要的现实感规则）：
   点击打工时喷向全屏的 `workFx` 粒子，必须是**所选工具会真实产生/喷出的东西**，
   而不是随便的图形。2~3 种造型都要呼应工具。对照参考：

   | 工具 | 真实喷出的粒子 |
   |---|---|
   | wateringCan 洒水壶 / waterCooler 饮水机 / mop 拖把 | 水滴 · 小水花 · 涟漪 |
   | wok 炒锅 / flatIron 熨斗 / shavedIce 刨冰机 | 蒸汽团 · 食材丁/冰屑 · 火星 |
   | mjolnir 雷神锤 / stormStaff 风暴杖 | 闪电折线 · 火花 · 电球 |
   | iceScepter 冰权杖 / snowGlobe 雪球 / acRemote 空调遥控 | 冰晶 · 雪花 · 冷雾 |
   | emberLantern 火种灯 / torch 料理喷枪 / skewer 烤串签 | 火苗 · 火星 · 青烟 |
   | fountainPen 钢笔 / tabletPen 数位笔 / printer 打印机 | 墨点 · 字符/符号 · 纸片 |
   | laptop 笔记本 / keyboard 机械键盘 | 代码括号 · 按键泡 · 火花 |
   | headset 耳机 | 音符 · 声波弧 |
   | crystalBall 水晶球 / laserPointer 激光笔 | 星芒 · 魔法光点 |
   | cauldron 药锅 | 气泡 · 魔法星 · 青烟 |
   | pondNet 捞网 / lifebuoy 救生圈 | 水花 · 泡泡 · 小鱼星 |
   | sproutWand 嫩芽杖 / yarnBall 毛线球 | 嫩叶/花瓣 · 绒球 |

   原则一句话：**先选工具，再让粒子成为"这个工具工作时飞出来的东西"**。冷工具冒冷雾、热工具冒蒸汽/火星、
   电工具迸电花、书写工具掉墨点/字符、水工具溅水滴。
7. **元素越多越华丽/越高阶**：物种拥有的元素越多，`decor` 装饰件（冠冕/宝石/环绕件/流苏…）
   和身体部件越多越繁复，一眼看出阶数高。单元素朴素，多元素盛装。
8. **主体 2~3 色分区，禁止单色调**（2026-07-16 用户评审加强）：accent/accent2 是**大面积第二、第三主色**
   （双色上下身/背腹异色/条纹/大斑块/帽壳异色——参考企鹅黑白肚、蜜蜂黄黑纹、舞狮红金白），不是小点缀；
   节点 fill 可直接写调色板外的 #rrggbb 补色。另：body 饱和、belly/脸明显更浅兜表情、忌灰扑扑。
9. **可靠性梯**（生成侧）：AI-rig 校验不过 → 退参数化 `bodyPlan` 动物体型 → 再退咕噜鸭；用最强模型（Opus）生成。
10. **真实原型锚定**（★2026-07-16 用户评审新增，对齐固有物种水准的关键）：每次生成**必须先选定一个
    真实存在的动物或植物原型**并写进必填字段 `prototype`（如"水獭"/"海象"/"捕蝇草"），整只设计照它的
    标志性特征刻画——路人一眼能说出"这是一只 XX"。**禁止无原型的抽象团子/几何拼块**。12 体型原型各带
    "原型候选"菜单（含植物：仙人掌/蘑菇墩/向日葵/蒲公英种子…），模型可选同类其它真实物种。
    校验层：customRig 路径缺 `prototype`（1~12 字）即拒绝。
11. **muzzle 面部刻画层 = 每只脸的个性**（★同上新增）：`RigViewParts.muzzle`（局部原点=脸中心，
    渲染在 head 之上、引擎眼嘴之下）**每只必画**，照原型画标志性五官周边——鸭扁嘴/鹰尖喙/猪鼻/
    海象白吻双獠牙/兔齿/熊猫眼斑/眉毛/胡须。引擎表情眼嘴叠画其上（嘴线落在喙/吻上），动画不受影响。
    同时脸参数随原型大胆变化：猛禽小珠眼 eyeR7~8、萌兽大圆眼 12~14、eyes 变体按性格。禁止全员同一张默认脸。
